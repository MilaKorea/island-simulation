# island simulation
Island Simulation

Описание
Island Simulation — консольное Java-приложение, моделирующее жизнь острова.
На острове есть животные и растения. Животные перемещаются, едят, размножаются, голодают и умирают.
Растения растут по шагам симуляции.

Как работает программа

При запуске пользователь вводит:
ширину острова
высоту острова
стартовое количество хищников
стартовое количество травоядных
стартовое количество растений
Создаётся остров из клеток (Cell).
Все животные и растения случайно распределяются по клеткам.
Запускается симуляция по шагам.

Что происходит на каждом шаге

Шаг животных:

Перемещение
Животное выбирает клетку в пределах своей скорости.
Движение только по вертикали и горизонтали.
Переход возможен только если в клетке есть место.
Питание
Хищники едят животных.
Травоядные едят растения.
Вероятность задаётся в YAML (food).

Размножение
Условия:

животное живо
не голодно
есть минимум 2 особи одного вида в клетке

Вероятность задаётся параметром reproductionChance.

Голод
Сытость уменьшается каждый шаг.
Если сытость ≤ 0 — животное умирает.
Удаление мёртвых
Мёртвые сущности удаляются из клеток.

Шаг растений:

В каждой клетке добавляется до plantGrowthPerStep растений,
если не превышен лимит на клетку.

Когда симуляция завершается

Симуляция останавливается, если:

достигнут максимум шагов (MAX_STEPS = 50)
или не осталось живых животных

Что вынесено в YAML

Файл: entity-config.yaml

В YAML находятся:

Параметры растений:

weight
maxPerCell

Параметры животных:

weight
maxPerCell
speed
foodRequired

Таблица питания:

кто кого ест
вероятность

Общие параметры:

reproductionChance
starvationRate
plantGrowthPerStep

Что задано в коде

В ApplicationRunner:

имя конфигурационного файла
максимальное количество шагов
период выполнения

В ConsoleInputHandler:
Рекомендованные диапазоны численности:

Хищники: 0.01 – 0.03
Травоядные: 0.03 – 0.08
Растения: 0.10 – 0.25

Используемые формулы

Максимум вида на острове:
globalMax = maxPerCell * количество клеток

Рекомендуемый диапазон:
min = globalMax * minCoeff
max = globalMax * maxCoeff

Голод:
сытость уменьшается на коэффициент starvationRate

Рост растений:
каждый шаг добавляется фиксированное количество растений

Структура проекта
